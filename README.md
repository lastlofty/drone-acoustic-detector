# Acoustic Drone Detector 🎤🛸

Учебный проект: **пассивное обнаружение дронов по звуку** в реальном времени.
Микрофон → лог-мел-спектрограмма → свёрточная сеть (CNN) → классификация
«дрон / фон» (легко расширяется на типы дронов).

Это **детектор, а не средство противодействия**: система только *слушает*
эфир и поднимает тревогу. Никакого глушения, перехвата или воздействия на
дрон — чистая задача распознавания сигнала (DSP + ML).

## Почему звук

Дальнобойные дроны, которые долетают до тыловых городов, часто «молчат» в
радиоэфире (автономная навигация), но имеют **характерный звук моторов** —
узнаваемый спектр жужжания винтов. Поэтому акустика — рабочий слой детекта
именно для этого класса угроз, и при этом дешёвый: нужен лишь микрофон.

## Стек

Python · PyTorch · librosa · sounddevice · soundfile

## Быстрый старт (проверка пайплайна за 2 минуты)

Без реальных записей — на синтетических данных, чтобы убедиться, что всё
работает end-to-end:

```bash
cd src
python make_synthetic_data.py   # генерит data/drone и data/background
python train.py                 # обучает CNN, сохраняет models/drone_cnn.pt
python detect.py                # слушает микрофон в реальном времени
```

В терминале появится «дашборд» с вероятностями и полоской тревоги:

```
[##############------] drone: 71%  background: 29%  <<< ДРОН!
```

### Веб-дашборд (опционально)

Тот же детектор, но в браузере — с живой спектрограммой, барами вероятностей
и тревожным статусом:

```bash
py -3.11 webapp.py
# открой http://127.0.0.1:8000
```

> ⚠️ На синтетике сеть учится отличать искусственный «буз» от шума — это
> демонстрация пайплайна, а не настоящий детектор. Для рабочей модели нужен
> **реальный датасет** (см. ниже).

## Реальный датасет

Структура простая: **каждая подпапка в `data/` = класс**.

```
data/
├── drone/        # записи дронов
└── background/   # тишина, улица, ветер, транспорт, голоса...
```

Можно добавить больше классов (например `drone_fpv`, `drone_shahed`) —
код подхватит их автоматически.

Два способа наполнить:

**1. Записать с микрофона** (есть готовый помощник):

```bash
python record.py drone 5 15        # 15 файлов по 5 сек с дроном
python record.py background 5 15    # фоновые записи
```

**2. Взять открытые датасеты**, например:
- поиск на Kaggle/GitHub по запросам *"drone audio dataset"*, *"UAV sound"*;
- **ESC-50** / **UrbanSound8K** — отличные источники для класса `background`
  (городские звуки, чтобы модель не путала дрон с мопедом/газонокосилкой).

Затем снова `python train.py` и `python detect.py`.

## Как это устроено

```
микрофон ─► окно 1 сек ─► мел-спектрограмма ─► CNN ─► softmax ─► тревога
            (скользящее)     (librosa)        (PyTorch)
```

| Файл                    | Назначение                                   |
|-------------------------|----------------------------------------------|
| `config.py`             | все параметры (частоты, окна, пути, порог)   |
| `features.py`           | аудио → нормализованная лог-мел-спектрограмма|
| `model.py`              | компактная CNN                               |
| `dataset.py`            | загрузка data/, нарезка на окна              |
| `make_synthetic_data.py`| генератор синтетики для проверки пайплайна   |
| `record.py`             | сбор реального датасета с микрофона          |
| `train.py`              | обучение + сохранение модели                 |
| `detect.py`             | детекция в реальном времени (терминал)       |
| `smoothing.py`          | сглаживание вероятностей во времени          |
| `webapp.py` + `static/` | веб-дашборд (FastAPI + WebSocket)            |

## Куда развивать

- ✅ **Сглаживание во времени** — реализовано (`smoothing.py`).
- ✅ **Веб-дашборд** — реализован (`webapp.py` + живая спектрограмма).
- **Несколько микрофонов + триангуляция** → не только «есть дрон», но и направление.
- **Тип дрона** по спектру (доп. классы вроде `drone_fpv`, `drone_shahed`).
- **Аугментация данных** (сдвиги, шум, реверберация) для устойчивости.

## Дисклеймер

Проект образовательный — для изучения DSP и ML на задаче распознавания
звука. Это пассивный датчик. Развёртывание систем мониторинга и
противодействия дронам регулируется законом и относится к компетенции
профильных госструктур.
